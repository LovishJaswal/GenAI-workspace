# Compiled vs Interpreted Languages

| Compiled Languages | Interpreted Languages |
|--------------------|----------------------|
| Compiled before execution | Executed by an interpreter |
| Most syntax errors are detected before execution | Errors are usually discovered while the program is running |
| Generally faster execution | Generally more flexible |
| Examples: C, C++ | Example: Python* |

> **Note**
>
> Python is often called an interpreted language, but internally it first compiles source code into **Bytecode**, which is then executed by the **Python Virtual Machine (PVM)**.

---

# Compile Time vs Run Time

## Compile Time

During compilation:

- Source code is analyzed.
- Syntax is checked.
- Bytecode is generated.

## Run Time

During execution:

- The Python Virtual Machine (PVM) executes the generated bytecode.
- Runtime errors (such as division by zero) may occur while the program is running.

> **Note**
>
> Unlike languages such as C or C++, Python performs compilation automatically before execution.

# Bytecode

Python does **not** execute `.py` files directly.

Instead, it first converts the source code into **Bytecode**, which is platform-independent.

The Python Virtual Machine (PVM) then executes this bytecode.

### Example

```python
print("Hello World")
```

Compiled Bytecode is typically stored inside the `__pycache__` directory.

---

# Python Virtual Machine (PVM)

The **Python Virtual Machine (PVM)** is the runtime engine responsible for executing Python bytecode.

Its responsibilities include:

- Executing bytecode instructions
- Managing memory
- Handling Python objects
- Managing function calls

---

# Viewing Bytecode using the `dis` Module

Python provides the built-in `dis` module to inspect the generated bytecode.

### Example

```python
import dis

def test():
    a = 5
    print(a)

dis.dis(test)
```

The `dis` module displays the bytecode instructions generated by Python.

This is useful for understanding how Python executes your code internally.

# Compiler vs Logic Errors

The Python compiler checks whether your code follows the correct syntax.

It **does not** check whether your logic is correct.

### Example

```python
age = 18

if age > 100:
    print("Eligible to vote")
```

This code is syntactically correct, but the logic is incorrect.

---

# Built-in Function Shadowing

Python allows built-in functions to be overridden (shadowed).

Example:

```python
print = 5
```

Although this is valid Python syntax, it should generally be avoided because the original `print()` function becomes inaccessible until the variable is removed.

> **Note**
>
> `print` is **not** a reserved keyword.
> It is a built-in function.

---

# Naming Best Practices

Avoid naming your Python files after built-in modules.

Examples:

- `math.py`
- `list.py`
- `set.py`
- `random.py`
- `string.py`
- `json.py`

These names can conflict with Python's standard library modules and may lead to import-related issues.

# Checking Data Types

The `type()` function returns the data type of an object.


# Type Conversion

Type conversion is the process of converting one data type into another.

## Implicit Type Conversion

Performed automatically by Python when it is safe to do so.

### Example

```python
a = 5
b = 2.5

print(a + b)
```

**Output**

```text
7.5
```

Python automatically converts `int` to `float`.

---

## Explicit Type Conversion

Performed manually by the programmer using built-in functions.

Common conversion functions:

- `int()`
- `float()`
- `str()`
- `bool()`
- `list()`
- `tuple()`
- `set()`
- `dict()` *(when applicable)*

### Example

```python
age = "20"

print(int(age))
```

# Mutable vs Immutable Objects

Understanding mutability is one of Python's most important concepts.

## Immutable Objects

Immutable objects **cannot be modified after they are created**.

Examples:

- `int`
- `float`
- `bool`
- `str`
- `tuple`
- `frozenset`
- `bytes`

---

## Mutable Objects

Mutable objects **can be modified after they are created**.

Examples:

- `list`
- `set`
- `dict`
- `bytearray`

---

# Why are Integers Immutable?

Instead of modifying an existing integer object, Python creates a **new object** whenever the value changes.

### Example

```python
a = 5
b = a

a = 10

print(a)
print(b)
```

### Output

```text
10
5
```

### Explanation

Initially:

```text
a в”Ђв”Ђв”Ђв–є 5
b в”Ђв”Ђв”Ђв–є 5
```

After executing:

```python
a = 10
```

Python creates a **new integer object**.

```text
a в”Ђв”Ђв”Ђв–є 10

b в”Ђв”Ђв”Ђв–є 5
```

The original integer object remains unchanged.

# Additional Notes

## Sets are Unordered

Sets do **not** support indexing because they are unordered collections.

Example:

```python
numbers = {10, 20, 30}

numbers[0]   # Error
```

If indexing is required, convert the set into a list first.
